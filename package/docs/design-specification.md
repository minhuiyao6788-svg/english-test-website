# 设计规范 - 英语等级测试网站

## 1. 设计方向与理念

**风格：** 教育友好风格（适龄调整版 15-60岁）

**视觉精髓：** 温暖的蓝绿色调营造轻松无压的测试环境，柔和圆角和充足留白降低焦虑感，彩色图标和微动画提供即时鼓励反馈。平衡友好亲和力与专业可信度，让用户在愉悦体验中完成英语水平评估。

**真实案例参考：** Khan Academy（友好教育界面）、Quizlet（学习卡片设计）、Duolingo（游戏化学习体验）、Coursera测验界面（专业但不严肃）

## 3. 组件规范

### 3.1 按钮（Button）

**结构：** 胶囊形状 + 中等字重文字 + 可选图标

**设计令牌应用：**
- 圆角：`radius-full`（999px）
- 内边距：`space-md` (16px) 垂直，`space-2xl` (32px-48px) 水平
- 字体：`button` (17px / 600 Semibold)
- 阴影：`shadow-card` 静态，`shadow-card-hover` 悬停
- 最小触摸区域：48×48px（移动端）

**变体：**

1. **Primary（主要按钮）**
   - 背景：`primary-500` → `primary-600` 悬停
   - 文字：`#FFFFFF`
   - 边框：2px solid `rgba(255,255,255,0.3)`
   - 悬停：`translateY(-2px)` + 阴影增强
   - 按下：`scale(0.97)`

2. **Secondary（次要按钮）**
   - 背景：`#FFFFFF` → `primary-50` 悬停
   - 文字：`primary-600`
   - 边框：2px solid `primary-500`
   - 悬停：`scale(1.02)` + 背景色变化

3. **Success（成功按钮）**
   - 背景：`success-500` → `success-600` 悬停
   - 文字：`#FFFFFF`
   - 用于提交答案、查看报告等确认操作

**状态反馈：**
- 禁用：50%透明度 + `cursor: not-allowed`
- 加载：按钮内旋转加载图标 + 文字淡化

**注意：** 仅使用 `transform` 和 `opacity` 动画，禁用 `width`/`height` 动画

### 3.2 卡片（Card）

**结构：** 白色背景 + 圆角 + 阴影 + 内边距

**设计令牌应用：**
- 背景：`background-card` (#FFFFFF)
- 圆角：`radius-lg` (16px) 标准卡片，`radius-2xl` (24px) Hero卡片
- 内边距：32px 四周
- 阴影：`shadow-card` 静态
- 边框：1px solid `neutral-200`（可选）

**变体：**

1. **测试类型卡片（可点击）**
   - 静态：白色背景 + `shadow-card`
   - 悬停：`translateY(-8px)` + `shadow-card-hover` + 2px solid `primary-500` 边框
   - 图标：64×64px 彩色线性图标居顶部
   - 标题：`h3` (24px / 600)
   - 描述：`body` (16px / 400)
   - 过渡：`duration-base` (250ms) + `easing-smooth`

2. **题目卡片（展示型）**
   - 大尺寸：最小高度320px
   - 题号：左上角小徽章（`primary-100` 背景 + `primary-700` 文字）
   - 题目：`body-lg` (18px / 400)，行高1.7
   - 选项：垂直排列，间距16px

3. **报告卡片（数据展示）**
   - 渐变背景：`linear-gradient(135deg, primary-50, neutral-50)`
   - 大数字：72px / 700 Bold，`primary-600` 颜色
   - 标签：`body-sm` (14px)，`neutral-400` 颜色

### 3.3 输入框（Input / Radio）

**单选选项设计：**

**结构：** 卡片式单选，整个区域可点击

**视觉方案：**
- 未选中：白色背景 + 2px solid `neutral-200` 边框 + `radius-md` (12px)
- 悬停：边框变为 `primary-500` + 轻微缩放 `scale(1.02)`
- 选中：2px solid `primary-500` 边框 + `primary-50` 背景 + 左侧彩色条（4px宽 `primary-500`）
- 内边距：16px 四周
- 最小高度：56px

**布局：**
- 选项标签：圆形单选圆点（左侧）+ 选项文字（`body` 16px）
- 垂直排列，间距12px

**动画：** 选中时淡入背景色 + 边框颜色变化（`duration-fast` 150ms）

### 3.4 导航栏（Navigation）

**桌面导航（顶部固定）：**

**结构：** 水平布局 + Logo左对齐 + 操作按钮右对齐

**设计令牌应用：**
- 高度：64px
- 背景：`#FFFFFF` + `shadow-sm`
- 内边距：0 32px
- Logo区域：品牌名称（`h3` 24px / 600）+ 可选图标
- 右侧：计时器组件 + 退出按钮

**移动导航（底部Tab）：**
- 高度：64px
- 背景：`#FFFFFF` + 顶部阴影
- 图标：24×24px + 标签（`body-sm` 14px）
- 激活状态：`primary-600` 颜色 + 图标放大 `scale(1.1)`

### 3.5 进度条（Progress）

**结构：** 顶部固定 + 平滑动画

**设计令牌应用：**
- 高度：6px
- 背景：`primary-100`
- 填充色：渐变 `linear-gradient(90deg, primary-500, primary-600)`
- 位置：页面最顶部，固定定位
- 圆角：`radius-full` 在右端
- 过渡：`width` 变化使用 `duration-base` (250ms) + `easing-smooth`

**文字指示：**
- 位置：进度条下方居中
- 文字：`body-sm` "15 / 50 题已完成"
- 颜色：`neutral-400`

**注意：** 进度条宽度变化使用 CSS `transition`，不是 `transform`（此为特殊情况）

### 3.6 计时器（Timer）

**结构：** 倒计时数字 + 时钟图标 + 视觉警告

**设计令牌应用：**
- 字体：`JetBrains Mono` 等宽字体，32px / 700 Bold
- 颜色：
  - 正常（>5分钟）：`neutral-700`
  - 警告（1-5分钟）：`warning-600`
  - 危险（<1分钟）：`error-500` + 轻微脉动动画
- 图标：24×24px 时钟图标（Lucide `Clock`）
- 布局：图标 + 数字水平排列，间距8px

**脉动动画（危险状态）：**
```
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.7; }
}
```
- 时长：1500ms
- 缓动：`linear`
- 循环：无限

## 4. 布局与响应式

**网站架构：** SPA单页应用（参考 content-structure-plan.md）

### 4.1 区域流程与视觉层级

**区域序列：** 首页Hero → 测试类型选择 → 测试进行页 → 测试报告页

**视觉层级：**

1. **首页（#home）**
   - Hero高度：450-500px
   - 主标题：`display` (56px) 居中
   - 副标题：`body-lg` (18px) 居中，`neutral-400` 颜色
   - CTA按钮：Primary大按钮（48px高），居中位置
   - 测试类型卡片：3列网格，间距24px

2. **测试进行页（#test）**
   - 顶部：固定导航（64px）+ 进度条（6px）
   - 题目区域：居中单列，最大宽度800px
   - 题目卡片：应用§3.2规范
   - 选项区域：应用§3.3规范
   - 底部操作：上一题/下一题按钮，间距16px

3. **测试报告页（#report）**
   - 分数展示：大卡片（应用§3.2报告卡片）居中，最大宽度600px
   - 雷达图：600px宽度居中
   - 错题列表：单列布局，每个错题卡片可展开
   - 建议区域：2列网格（桌面）→ 1列（移动）

**导航模式：** 固定顶部导航 + 路由切换（无刷新）

**过渡效果：** 页面切换使用淡入淡出（`duration-slow` 350ms）

### 4.2 网格与容器策略

**断点定义：**
- `sm`: 640px（手机横屏）
- `md`: 768px（平板竖屏）
- `lg`: 1024px（平板横屏/小笔记本）
- `xl`: 1280px（桌面）

**容器最大宽度：**
- 内容容器：1200px（居中）
- 题目区域：800px（窄容器，提高可读性）
- 报告主区域：1000px

**网格系统：**
- 测试类型卡片：3列（桌面）→ 2列（平板）→ 1列（移动）
- 列间距：24px（桌面）→ 16px（移动）
- 建议卡片：2列（桌面）→ 1列（移动）

**内边距：**
- 页面边距：32px（桌面）→ 20px（移动）
- Hero内边距：`space-4xl` (96px) 垂直（桌面）→ `space-3xl` (64px)（移动）

### 4.3 响应式适配

**移动端优化（<768px）：**
- 标题尺寸缩小（参考§2.2字阶移动尺寸）
- 卡片内边距：32px → 24px
- 测试类型卡片：1列堆叠
- 按钮：全宽度（除导航栏内小按钮）
- 底部Tab导航替代顶部导航（可选）
- 题目选项：保持垂直排列，增大触摸区域至56px高

**平板适配（768px-1024px）：**
- 测试类型卡片：2列网格
- Hero区域：维持桌面布局，适度缩小间距
- 题目区域：居中，最大宽度700px

**触摸优化：**
- 所有交互元素最小触摸区域：48×48px
- 按钮内边距增加（移动端）
- 选项卡片高度增加至56px

## 5. 交互与动画

**动画标准：**
- 时长：参考§2.6动画系统
- 缓动：默认 `easing-smooth`，庆祝场景 `easing-bounce`
- 仅动画化 `transform` 和 `opacity`（性能优先）

**组件交互动画：**

1. **按钮**
   - 悬停：`translateY(-2px)` + `shadow-card-hover`（250ms）
   - 按下：`scale(0.97)`（150ms）

2. **卡片（测试类型）**
   - 悬停：`translateY(-8px)` + `shadow-card-hover` + 边框变色（250ms）

3. **选项卡片**
   - 悬停：`scale(1.02)` + 边框变色（150ms）
   - 选中：背景色淡入 + 左侧彩色条滑入（150ms）

4. **进度条**
   - 填充动画：宽度变化（250ms `easing-smooth`）

5. **计时器危险状态**
   - 脉动动画：1500ms 无限循环

**页面过渡：**
- 路由切换：淡出（200ms）→ 淡入（350ms）
- 模态框：缩放 `scale(0.95 → 1)` + 淡入（350ms `easing-smooth`）

**庆祝动画（提交测试时）：**
- 按钮变形：宽度扩展 + 文字"提交中..." + 旋转图标（600ms `easing-bounce`）
- 完成后跳转到报告页

**进度庆祝（每完成10题）：**
- 进度条填充色：渐变闪烁（500ms，一次）
- 提示文字：短暂缩放 `scale(1.1)` + 颜色变为 `success-500`（500ms）

**无障碍支持：**
- 支持 `prefers-reduced-motion`：关闭所有非必要动画
- 保留必要反馈（选中状态颜色变化、禁用状态）

## 6. 视觉元素与图标

**图标系统：**
- 图标库：Lucide Icons（SVG线性风格）
- 尺寸：16px（小）、24px（标准）、32px（大）、64px（卡片装饰）
- 颜色：继承父元素或独立设置（`primary-500`/`neutral-400`）
- 推荐图标：
  - 测试类型：`BookOpen`（四六级）、`Plane`（托福）、`Globe`（雅思）
  - 操作：`Play`（开始）、`ChevronLeft/Right`（导航）、`Check`（提交）
  - 状态：`Clock`（计时）、`CheckCircle`（正确）、`XCircle`（错误）

**轻量插图使用：**
- 位置：Hero区域右侧装饰、测试报告页顶部
- 风格：线性插图（单色或双色调），简约几何风格
- 尺寸：300-400px宽度
- 推荐主题：
  - 学习场景：看书、戴耳机的人物
  - 抽象概念：上升箭头、星星、奖杯
- 获取方式：Undraw（推荐）、Storyset、或自定义SVG

**彩色图标（测试类型卡片）：**
- 风格：线性图标 + 圆形彩色背景
- 背景色：
  - 四六级：`primary-100` 背景 + `primary-600` 图标
  - 托福：`#E0F2FE` 背景 + `#0284C7` 图标
  - 雅思：`#F0FDF4` 背景 + `#16A34A` 图标
- 尺寸：64×64px 圆形背景 + 32×32px 图标

**装饰性背景元素：**
- Hero区域：轻微渐变背景（`neutral-50` → `primary-50`，5%透明度）
- 可选：右上角/左下角几何形状装饰（圆形/圆角矩形，10%透明度）

**禁止使用：**
- ❌ Emoji作为UI图标（仅可用于用户内容示例）
- ❌ 过于复杂的插图（保持简约）
- ❌ 卡通风格（目标用户15-60岁，避免过于幼稚）

## 7. 温馨文案风格指南

**文案原则：**
- **鼓励性** - 使用积极正面的词汇，避免"错误"、"失败"等负面词
- **口语化** - 自然对话感，避免生硬的官方语气
- **简洁明了** - 每句话不超过15字（中文），避免长句
- **适度幽默** - 可以使用轻松的比喻，但不过度搞笑
- **年龄适配** - 15-60岁都能理解和接受的表达方式

**文案示例：**

| 场景 | 常规表达 | 温馨表达 ✅ |
|------|---------|------------|
| 欢迎语 | 欢迎使用英语测试系统 | 嗨！准备好测一测英语水平了吗？ |
| 选择提示 | 请选择测试类型 | 选择一个测试，开启你的英语探索之旅 |
| 计时提示 | 剩余时间不足 | 加油！还剩一点时间，你可以的 💪 |
| 答题鼓励 | 已完成50% | 太棒了！已经完成一半啦，继续保持 🎉 |
| 提交确认 | 确认提交答案？ | 答完啦？点击查看你的专属报告吧 |
| 错题反馈 | 答案错误 | 这道题可以再想想，正确答案是... |
| 完成祝贺 | 测试已完成 | 恭喜完成测试！来看看你的成绩单 🎊 |
| 建议语 | 建议加强语法练习 | 多做些语法练习，进步会更快哦 📚 |

**按钮文案：**
- 开始测试 → "开始挑战"
- 下一题 → "下一题"（保持简洁）
- 提交答案 → "查看我的成绩"
- 返回首页 → "再测一次"

**提示信息：**
- 加载中："正在加载题目，马上就好..."
- 保存成功："答案已记录 ✓"
- 时间到："时间到啦！来看看你的答题情况"

**注意事项：**
- Emoji使用：每段文案最多1个，仅用于情感表达（✅ 可用），不用于功能图标（❌ 禁止）
- 避免过度口语化："嘞"、"啦"、"呢"等语气词适度使用，每句最多1个
- 错误提示温和化：不说"错误"，改为"可以再想想"、"这次没选对"
