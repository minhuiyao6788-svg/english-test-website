# 托福TOEFL测试题库创建蓝图

## 执行摘要与目标界定

本蓝图旨在为教研团队、题库编辑与质量审核人员提供一套可落地、可复用的托福(TOEFL iBT)题库建设方案,覆盖基础版(50题)与完整版(100题)两个版本,全部统一为四选一单选题,并配套标准答案与详细解析。题库将保存为单一JSON文件(data/toefl_questions.json),以清晰的字段与层级结构承载版本、分类、难度、知识点标签与解析模板等元数据,确保后续教学产品与测评系统的无缝对接与扩展。

为保证与真实考试的贴合度,蓝图严格对齐托福阅读(Reading)部分的官方说明,包括学术文章来源、题量与时间、题型定义与能力要求等;同时,针对非官方渠道常见的“语法题”提法,本方案明确在iBT体系中将其统摄于阅读能力框架下的“句子简化(Sentence Simplification)”与“句法结构/书面表达(Structure & Written Expression, ITP遗留范式)”两类,以保障术语与能力指向的一致性与可验证性[^1][^2][^3][^8]。

成功标准包括:
- 数量与结构:基础版50题(词汇15、阅读中的词汇/细节/推断/目的/简化/插入等共35)、完整版100题(词汇30、阅读70),总计150题。
- 题型与选项:全部为四选一单选题;阅读题覆盖官方阅读题型的核心子集;词汇题覆盖单词与短语两类。
- 难度与分布:基础版以中低难度为主,完整版覆盖中档至中高难度;难度标签与知识点标签体系化。
- 解析质量:答案确定、错误项可解释、策略可迁移;与官方题型能力描述对齐[^1][^2][^6]。
- 合规性:不包含受版权保护的真题原文;段落与题干原创或改写;素材来源与版权标注清晰。

信息缺口说明:
- ETS官方阅读材料受版权保护,题库不得直接使用原文;需采用原创/改写素材并明确版权标注[^2]。
- “语法题”需明确为iBT体系中的句子简化与(在iBT中不存在的)结构/书面表达题型之区分与映射;本蓝图统一采用阅读框架下的“句子简化”作为语法能力承载[^1][^8]。
- 难度等级与题目标注的量化标准需结合官方能力描述与行业经验制定,并在试测后迭代优化[^6]。

## 托福考试结构与题型综述(对齐官方)

托福iBT阅读部分聚焦学术环境下的阅读理解能力,官方说明指出:每场考试包含两篇阅读文章,每篇约700词,每篇约10题,总计约20题,建议用时约35分钟。文章节选自大学程度的教科书,用于主题导入,覆盖多学科主题;阅读中提供术语表(Glossary)功能以解释不常见词汇[^1]。官方样题文档亦强调练习题与2023年7月26日及之后实施的考试一致,同时提示某些题型可能不出现在特定可及性适配版本中[^2]。

题型范围(阅读)包括:事实信息与负事实信息、推理与修辞目的、词汇(词/短语在语境中的含义)、句子简化、文本插入、篇章总结等。这些题型共同评估考生对信息识别与理解、语义推断、作者意图把握、句法改写与信息整合等能力[^1][^2][^4][^5]。

为使读者直观把握阅读部分的组成与节奏,以下表格概述关键参数与题型映射。

表1:阅读部分关键参数一览(官方对齐)

| 指标 | 数值/说明 |
|---|---|
| 文章数量 | 2篇 |
| 文章长度 | 每篇约700词 |
| 题量 | 每篇约10题,总计约20题 |
| 时间 | 约35分钟 |
| 文章来源 | 大学程度教科书的主题导入段落 |
| 题型范围 | 事实/负事实信息、推理/修辞目的、词汇、句子简化、文本插入、篇章总结 |
| 辅助功能 | Glossary(术语表)用于解释不常见词汇 |

上述参数为题库设计与编排提供现实约束:题库在模拟真实考试节奏时,应控制阅读篇章长度与题量分布,保证训练与真实体验的一致性[^1][^2]。

表2:阅读题型与能力要求映射(官方对齐)

| 题型 | 能力要点 | 典型设问 |
|---|---|---|
| 事实信息 | 识别并理解文中直接陈述的信息 | Which of the following is stated about X? |
| 负事实信息 | 辨识文中未出现或与事实相悖的信息 | Which of the following is NOT mentioned about X? |
| 推理 | 基于文本线索进行逻辑推断 | What can be inferred about X? |
| 修辞目的 | 理解作者为何提及某信息 | Why does the author mention X? |
| 词汇(词/短语) | 在语境中判断词义/短语义 | The word/phrase “X” in the passage is closest in meaning to |
| 句子简化 | 提取复杂句核心命题,选择同义简化句 | Which of the following best restates the sentence? |
| 文本插入 | 判断句子在段落中的最佳位置 | Where does the sentence best fit? |
| 篇章总结 | 选择三个最能概括主旨的陈述 | Select three statements that best describe the main ideas |

上述映射用于指导题目创作与标签体系,确保能力指向清晰且与官方描述一致[^1][^2][^4][^5]。

## 题目分类与数量配置(基础版与完整版)

为满足教学与测评的渐进式需求,题库分为基础版与完整版两个版本,全部统一为四选一单选题。分类与数量配置如下:

- 基础版(50题)

  - 词汇题:30(单词20、短语10)
  - 阅读理解题:70(覆盖与基础版相同题型范围,并提升推断与修辞目的占比)

## 难度等级与标签体系设计

难度等级采用“基础/中级/进阶/高阶”四级,结合官方能力描述(Performance Descriptors)进行界定。官方描述指出不同分数段考生在阅读中的理解层次差异,如低中级(Low-Intermediate)通常能理解部分主旨与重要信息,高级(Advanced)能够较为全面与准确地把握论证结构与细节逻辑[^6]。据此,本方案将难度与题型能力要求对齐,形成可操作的标注规则。

表5:难度等级定义与示例(结合官方能力描述)

| 难度 | 能力界定(阅读) | 适配题型 | 示例特征 |
|---|---|---|---|
| 基础 | 能识别直接陈述的主旨与明显信息;对局部语义关系把握有限 | 事实信息、基础词汇 | 明确线索、单一线索定位、常见学术词汇 |
| 中级 | 能理解段落主旨与重要细节;能进行简单推断 | 事实/负事实、推理入门、句子简化基础 | 同义改写、简单逻辑关系、常见连接词 |
| 进阶 | 能把握隐含信息与作者意图;能解析复杂句法 | 推理/修辞目的、句子简化进阶、文本插入 | 跨句衔接、复杂从句、语气与态度判断 |
| 高阶 | 能整合跨段信息与论证结构;能处理抽象与专业话题 | 推理综合、文本插入、篇章总结 | 论证框架识别、主题升华、学科术语语境化 |

知识点标签体系用于精细化学习路径与练习组织,覆盖学科主题、语法/句法、阅读技能与语言特征四个维度。

表6:知识点标签体系与示例

| 标签维度 | 标签值示例 | 说明 |
|---|---|---|
| 学科主题 | 生物学、生态学、天文学、地质学、经济学、心理学、艺术史 | 与文章主题对应,支持主题化练习 |
| 语法/句法 | 时态/语态、主谓一致、连接词(因果/转折/对比)、从句(定语/状语)、非谓语、平行结构 | 与句子简化、文本插入能力直接相关 |
| 阅读技能 | 事实/负事实、推断、修辞目的、词汇、句子简化、文本插入、篇章总结 | 题型能力标签,支撑策略训练 |
| 语言特征 | 学术词汇、专业术语、名词化、被动语态、复杂句 | 支撑词汇题与句法解析训练 |

难度标注规则:
- 基础:识别直接信息、常见词汇、单一逻辑关系。
- 中级:同义改写、段落主旨、简单推断。
- 进阶:隐含信息、复杂句法、修辞目的、跨句衔接。
- 高阶:跨段整合、论证结构把握、抽象主题处理。

上述定义与标注规则与官方能力描述保持一致性,并通过试测数据迭代优化[^6]。

## 词汇题设计规范(语境化与干扰项质量)

词汇题以语境化为核心,要求在给定学术或准学术段落中判断词/短语在上下文中的最佳含义。设问遵循官方与常见样题的表述,如“The word/phrase ‘X’ in the passage is closest in meaning to”,并提供四个选项。正确项应与语境高度一致;干扰项基于常见误配语义、词形相近(形近但义远)、广义/狭义过泛或与主题无关的语义牵引进行设计,确保区分度与可解释性[^1][^2][^4][^5]。

表7:词汇题干扰项设计原则与示例

| 设计原则 | 说明 | 示例(示意) |
|---|---|---|
| 语境一致性 | 正确项必须与上下文逻辑与语义契合 | 段落讨论“季节性迁徙”,“breeding”应指向“繁殖”而非“育种” |
| 语义过泛 | 干扰项选取更宽泛或无关含义 | “model”若指“模型”,干扰项用“模特”(词形无关,语义过泛) |
| 形近误配 | 利用词形相近制造干扰 | “economic”与“economical”混用,一个“经济上的”,一个“节俭的” |
| 主题牵引 | 用段落主题的常见搭配误导 | 生态段落中“impact”易被理解为“正面影响”,实际为“影响”(中性) |
| 搭配差异 | 利用固定搭配差异设置陷阱 | “draw conclusions”误配为“描绘结论”(动词搭配误用) |

词汇来源以学术与通用学术词汇为主,避免生僻专业术语;当术语出现时,题干应提供足够语境线索以支持推断[^1][^2]。

## 语法题呈现策略(与TOEFL iBT对齐)

在iBT体系中,传统意义上的“语法题”并不作为独立计分题型存在;其能力要求主要通过阅读中的“句子简化(Sentence Simplification)”来考查,要求考生选择与原句同义的简化版本,从而检验对复杂句法、逻辑关系与核心命题的把握[^1]。另一方面,TOEFL ITP中的“Structure and Written Expression”样题体现了对时态、主谓一致、介词与连词等语法点的系统考查,可作为语法知识点清单与干扰项设计的参考来源,但在iBT题库中需以阅读能力呈现,避免与独立语法题型混淆[^8]。

表8:语法知识点清单与题型映射

| 语法知识点 | 典型错误 | 题型承载(iBT) | 干扰项设计要点 |
|---|---|---|---|
| 时态/语态 | 过去/完成混用;主动/被动误配 | 句子简化 | 提供时态不一致的选项;被动语义误置 |
| 主谓一致 | 单复数误判;倒装一致性缺失 | 句子简化 | 主语与动词数不一致选项 |
| 介词搭配 | 固定搭配错误 | 句子简化 | 与动词/名词搭配不当的介词选项 |
| 连词/逻辑关系 | 因果/转折/对比混淆 | 句子简化、文本插入 | 错误逻辑关系的选项(伪因果、伪转折) |
| 从句结构 | 定语从句与状语从句混用 | 句子简化 | 关系词误用、从句残缺 |
| 非谓语与平行结构 | 分词与不定式误用;平行不一致 | 句子简化 | 非谓语逻辑主语不一致;平行结构失衡 |

本方案将“语法题”统一在阅读框架下呈现:以句子简化与文本插入为主,兼顾对语法点的精准考查与干扰项的可解释性[^1][^8]。

## 阅读理解题设计规范(题型覆盖与篇章主题)

阅读理解题覆盖官方题型核心子集:事实/负事实信息、推理/修辞目的、句子简化、文本插入与篇章总结(以单选题形式呈现)。篇章主题建议涵盖理工、人文与社科,长度控制在约700词级别,结构清晰(总分总、对比、因果、问题-解决等),以保证与官方文章类型与长度的一致性[^1][^2]。

表9:阅读题型-设问模板-能力目标对照表

| 题型 | 设问模板(示意) | 能力目标 |
|---|---|---|
| 事实信息 | According to the passage, which of the following is true about X? | 直接信息识别与定位 |
| 负事实信息 | Which of the following is NOT mentioned about X? | 排除与辨析能力 |
| 推理 | Which of the following can be inferred about X from the passage? | 基于证据的合理推断 |
| 修辞目的 | Why does the author mention X? | 作者意图与论证理解 |
| 句子简化 | Which of the following best restates the sentence “…”? | 句法改写与核心命题提取 |
| 文本插入 | Where does the sentence best fit in the paragraph? | 衔接与信息结构把握 |
| 篇章总结 | Which of the following best summarizes the main idea of the passage? | 主旨概括与信息整合 |

篇章结构建议:
- 主题导入(背景与问题提出)
- 论证展开(对比/因果/例证)
- 小结与延伸(结论与启示)

长度控制与难度分配:
- 基础版:更多事实信息与基础词汇,少量简单推断与句子简化。
- 完整版:提升推断与修辞目的比例,增加句子简化与文本插入的复杂度。

上述规范与官方题型说明保持一致,并参考常见题型的设问方式与策略文章进行细化[^1][^2][^4][^5]。

## 答案与解析模板(可解释性与策略化)

解析需同时满足“答案确定”“错误项可解释”“策略可迁移”三项要求。解析结构建议如下:
- 答案:明确选项(A/B/C/D)。
- 定位线索:指出原文中支持答案的关键词句或逻辑关系。
- 干扰项分析:逐项解释错误原因(语义过泛、逻辑不符、搭配不当等)。
- 题型策略:给出该题型的通用解题路径(如定位—同义改写—排除)。
- 知识点标签:关联学科主题与语法/句法、阅读技能标签。
- 难度评估:说明难度判定理由(线索复杂度、推断深度、句法难度等)。

表10:解析模板字段说明与示例(示意)

| 字段 | 内容要点 | 示例(示意) |
|---|---|---|
| 答案 | 明确正确选项 | B |
| 定位线索 | 原文关键句/逻辑词 | “In contrast…”提示对比关系 |
| 干扰项分析 | 错误原因逐项解释 | A:语义过泛;C:逻辑方向错误;D:搭配不当 |
| 题型策略 | 通用方法与步骤 | 事实信息题:定位→同义改写→排除 |
| 知识点标签 | 学科/语法/技能维度 | 生态学;连接词(对比);事实信息 |
| 难度评估 | 线索与推断复杂度 | 中级:单一线索,明显同义改写 |

解析质量标准:
- 正确项有据可依;错误项可解释且不牵强附会。
- 策略提炼可迁移至同类题型。
- 与官方能力描述与题型定义对齐(词汇、句子简化、推理与修辞目的等)[^1][^2]。

## JSON元数据与文件结构规范(data/toefl_questions.json)

为实现跨系统兼容与扩展性,JSON文件采用清晰的字段与层级设计。建议顶层结构如下:
- version:版本标识(basic/full)
- generated_at:生成时间(ISO 8601)
- metadata:元数据
- questions:题目数组

metadata建议字段:
- total:总题量
- categories:分类统计(vocabulary/reading)
- difficulty_distribution:难度分布(基础/中级/进阶/高阶)
- tags_vocabulary:词汇标签集合
- tags_grammar:语法标签集合
- tags_reading_skills:阅读技能标签集合
- subjects:学科主题集合

question对象字段:
- id:题目唯一标识
- type:vocabulary/reading
- subtype:词汇(单词/短语)、阅读题型(事实/负事实/推理/修辞目的/句子简化/文本插入/总结)
- passage_id:篇章标识(阅读题)
- passage_title:篇章标题

## 质量保障与合规性(版权与真实性)

质量保障贯穿创作、审核与试测全流程:
- 原创与改写:所有段落与题干需原创或基于公开素材的改写,避免使用受版权保护的官方真题原文;source_note字段明确素材来源与版权标注[^2]。
- 难度与区分度:依据难度标注规则与题型能力要求进行初稿标注;组织小样本试测,基于正确率与用时等指标迭代优化难度与干扰项。
- 审核流程:教研、编辑与审核三轨并行;交叉审题确保答案与解析的准确性、逻辑性与可迁移性。
- 真实性:长度、题型与设问风格对齐官方说明与样题;词汇与语法点覆盖学术场景常见用法[^1][^2]。

## 制作流程与交付计划

流程分阶段推进,确保质量与效率:
1. 规划与设计:完成题型规范、标签体系与JSON结构。
2. 命题:按版本与分类配额生成初稿(基础版→完整版)。
3. 审核:交叉审题与难度校准。
4. 试测:小样本试测与数据分析。
5. 修订:基于试测结果优化难度与干扰项。
6. 定稿与交付:生成JSON文件并完成元数据与标签。

## 附录:参考资料与对齐说明

本蓝图严格对齐托福iBT阅读部分的官方说明与样题,并参考第三方题型说明与策略文章,以确保题型定义、能力指向与设问风格的真实性与可操作性。所有题型与篇章长度、时间与题量建议均以官方页面与样题文档为准;第三方资料用于补充题型细目与常见设问模式。版权与使用限制方面,明确避免使用受保护的官方真题原文,采用原创或改写素材,并在source_note中标注来源与版权说明。

信息缺口与处理:
- 语法题定位:在iBT体系中以阅读的句子简化为主,避免独立语法题型;语法知识点清单参考ITP样题,但在题库中以阅读能力呈现[^1][^8]。
- 难度量化:结合官方能力描述与行业经验制定初始标准,并在试测后迭代优化[^6]。
